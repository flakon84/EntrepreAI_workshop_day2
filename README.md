# EntrepreAI workshop — process repo

## Главное правило
**Любая работа в этом репозитории (код, документы, решения, эксперименты) выполняется по процессу из `docs/00-howto.md`.**

Если ты начинаешь задачу/обсуждение и не можешь указать `Trace` до `docs/10-vision.txt` — значит мы работаем «вслепую» и сначала надо восстановить трассировку или оформить отклонение.

- HOWTO (обязательно прочитать/следовать): `docs/00-howto.md`
- Vision & Values (точка истины сверху): `docs/10-vision.txt`

## Как пользоваться (коротко)
1) Перед любой значимой работой делаем **Pre‑flight** по `docs/00-howto.md`.
2) В PR/issue/документах добавляем:
   - `Trace: VIS-xxx -> STR-xxx -> RM-xxx -> PRD-xxx -> ADR-xxx`
   - `Evidence: ...`
3) Если надо отступить от норм — используем **Deviation protocol** (описан в HOWTO).

## Типовой сценарий №1 (фиксируем как «дефолтный путь»)
От ценностей к типам деятельности -> стратегии -> роадмапу -> плану -> требованиям к продуктам -> упаковке в компанию/форму.

### Выходы сценария (что должно появиться)
- `docs/10-vision.txt` — Vision & Values (`VIS-###`).
- `docs/20-activity-map.md` — типы деятельности и артефакты.
- `docs/30-strategy.md` — стратегия (`STR-###`).
- `docs/40-roadmap.md` — roadmap (`RM-###`).
- `docs/50-plan.md` — план на ближайший период.
- `docs/70-prd/PRD-###.md` — PRD инициатив.
- `docs/60-company-packaging.md` — упаковка в компанию/форму.

### Мини‑правило
Если мы не можем показать эти файлы (хотя бы в черновом виде) — значит мы не «строим систему», а делаем разрозненные действия.

## Старт: с чего начинаем сегодня
1) Заполняем `docs/10-vision.txt` (ценности + non‑negotiables + нумерация `VIS-###`).
2) Затем создаем `docs/20-activity-map.md` и связываем с Vision.
3) Потом — `docs/30-strategy.md`, `docs/40-roadmap.md`, и только после этого — PRD.
