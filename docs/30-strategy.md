# Strategy

Trace: VIS-001 -> VIS-010 -> VIS-011 -> VIS-040

## 1) Контекст (что мы строим)
Мы строим деятельность на пересечении:
- **локального STEM/инженерного сообщества (Мордовия)** и
- **практической IT‑помощи предпринимателям** (код, автоматизация, прототипирование).

Стратегический критерий (VIS-040): **не делаем работу ради работы** — делаем только то, что дает **социальный эффект** и может быть **масштабировано**.

## 2) Стратегическая гипотеза (черновик)
Если мы создадим:
1) локальный «физический» вход (инженерный кружок),
2) медиа‑контур доверия и дистрибуции (подкаст/радио),
3) продуктовый контур масштабирования (приложение для совместного инженерного творчества),
то получим маховик: **люди -> проекты -> истории/контент -> новые люди/партнеры -> инструменты -> больше проектов**.

## 3) Стратегические ставки (STR)
Нумерация с шагом 10 — чтобы можно было вставлять промежуточные ставки.

### STR-010: Новый тип инженерного кружка (радио/робототехника)
- Почему это ставка:
  - максимальный локальный социальный эффект (VIS-010),
  - быстрые «осязаемые» результаты (прототипы),
  - генератор проектов/кейсов для медиа и продукта.
- Что должно быть правдой (assumptions):
  - есть критическая масса участников (ядро 10–20 человек),
  - есть место/партнер (школа/вуз/кружок/фаблаб),
  - есть формат занятий, который дает результат за 4–6 встреч.
- Evidence (как проверим быстро):
  - пилот 4 недели: 4–6 встреч, 1–2 законченных мини‑проекта,
  - список ожидания/заявок (N),
  - ретеншн (сколько пришли повторно).
- Kill‑criteria (когда закрываем/меняем):
  - нет ядра участников после 2–3 попыток набора,
  - не можем довести до демонстрируемого результата.

### STR-020: Радио / подкаст (контент как дистрибуция и доверие)
- Почему это ставка:
  - масштабирование влияния без географических ограничений,
  - привлечение менторов/партнеров/участников,
  - упаковка смысла и кейсов, рост доверия.
- Assumptions:
  - есть регулярность (хотя бы 1 выпуск/неделю),
  - есть гости/темы из реальных проектов,
  - контент приводит конкретные действия (подписки, заявки, партнерства).
- Evidence:
  - 6 выпусков за 6 недель,
  - 3–5 входящих контактов/неделю после 3–4 выпуска,
  - конверсия в участие/партнерство.
- Kill‑criteria:
  - не получается держать ритм,
  - нет понятных сигналов полезности (входящих/обратной связи).

### STR-030: Приложение для совместного инженерного творчества
- Почему это ставка:
  - технологическое масштабирование кружка/сообщества,
  - инфраструктура для совместных проектов (процесс + артефакты + доказательства),
  - потенциал продукта/монетизации без «работы ради работы».
- Assumptions:
  - есть повторяемый workflow у участников (проект, задачи, артефакты, доказательства),
  - людям нужна совместная работа/трекшн, а не просто чат,
  - мы можем сделать MVP, который реально используют.
- Evidence:
  - не начинать с “полного продукта”: сначала прототип на 1 workflow,
  - 10 активных пользователей, 3 активных проекта, удержание 2 недели,
  - измеримые артефакты (проекты/документация/демо).
- Kill‑criteria:
  - нет регулярного использования даже в «своем» кружке,
  - основная ценность решается существующими инструментами без потерь.

## 4) Фокус и анти‑фокус
### Фокус (делаем)
- Быстрые циклы проверки (evidence-first).
- Результаты, которые можно показать (демо, прототипы, истории).
- Маховик: кружок -> проекты -> контент -> рост сообщества -> инструменты.

### Анти‑фокус (сознательно не делаем)
- «Большая разработка приложения» до подтвержденного workflow и спроса.
- Контент «ради контента» без связи с проектами/сообществом.
- Мероприятия без результата/демо и без повторяемого формата.

## 5) Метрики успеха (черновик)
### Leading indicators (ранние)
- # активных участников/неделю (кружок)
- # завершенных мини‑проектов/месяц
- # входящих запросов/партнерств из подкаста
- # активных проектов и удержание в MVP (если делаем)

### Lagging indicators (поздние)
- рост сообщества (ядро/менторы/партнеры)
- социальный эффект (конкретные проекты/результаты участников)
- масштабирование формата (повторяемость, новые площадки)
