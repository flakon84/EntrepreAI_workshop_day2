# Staffing — штатное расписание (пилот PRD-010)

Trace: VIS-001 -> VIS-010 -> VIS-011 -> VIS-040 -> STR-020 -> PRD-010
Evidence: ритм 6/6 выпусков + actions из weekly-log

## Принцип
Для пилота мы оптимизируем под:
- регулярность,
- минимальные затраты времени/денег,
- понятный раздел труда.

Штатное расписание ниже — это роли (функции). Их можно совмещать одним человеком.

## Вариант A — Solo (минимальный)
Один человек совмещает все роли.
- Продюсер/ведущий (owner): 6–10 ч/нед
- Постпродакшн (монтаж): 2–6 ч/нед
- Дистрибуция + комьюнити: 2–4 ч/нед
Итого: 10–20 ч/нед (зависит от монтажа и наличия гостей).

## Вариант B — Lean team (рекомендуемый, если есть бюджет)
### 1) Продюсер/ведущий (1 человек)
- Нагрузка: 6–10 ч/нед
- Зона ответственности: тема/гость, “зачем выпуск”, запись, качество, CTA.

### 2) Редактор/монтажёр (0.25–0.5 ставки)
- Нагрузка: 3–8 ч/нед
- Зона ответственности: звук/монтаж, экспорт, передача файлов, тех. контроль.

### 3) Дистрибуция/комьюнити (0.25 ставки)
- Нагрузка: 2–4 ч/нед
- Зона ответственности: публикация, посты в TG/VK, сбор вопросов/реакций, клипы (если делаем).

## Вариант C — Growth (после пилота или если быстро пошло)
Добавляется:
- Ресерчер/букер гостей (0.25 ставки): чтобы всегда была очередь тем/гостей.
- Дизайнер (по задачам): обложка, шаблоны, визуалы.

## Порядок усиления (если упираемся в предел)
1) Сначала — монтажёр (снимает главный bottleneck по времени).
2) Затем — букер/ресерчер (стабилизирует поток гостей и тем).
3) Затем — комьюнити/дистрибуция (увеличивает actions и обратную связь).

## KPI / критерии результативности по ролям (пилот)
- Продюсер/ведущий: 6/6 выпусков, у каждого “зачем” + CTA, нет срывов публикации.
- Монтажёр: turnaround <= 48 часов после записи; стабильный уровень звука; нет тех. брака.
- Дистрибуция/комьюнити: посты/публикации без пропусков; собраны реакции и отражены в weekly-log.
